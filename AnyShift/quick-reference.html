Operating Modes

Anyshift supports two fundamentally different ways of communicating your gear selection to the game. You select the mode per profile.

Simulated Keystrokes Mode

This is the default mode. When you engage a gear on your H-shifter, Anyshift figures out how many sequential up- or downshifts are needed to reach that gear from the current one, and sends the corresponding number of key presses to the game.

Example: Currently in 2nd gear and you slot into 4th — Anyshift sends two S (upshift) key presses.

ℹ How to use
Set the exact same keys in Anyshift's Key Selection box and in your game's controller/keyboard settings for upshift and downshift.

Occasionally the game and Anyshift can get out of sync (e.g. on rolling starts). You can re-sync by manually pressing the keyboard keys to match the gear shown in-game.

Memory Write Mode

In this mode, Anyshift writes the gear number directly to the game's memory. The game always reflects exactly what is on the shifter — perfect sync, no key presses needed.

Requires knowing the game's memory address for the gear variable. You can find it using Cheat Engine. Several pre-configured presets are included (GT4, Enthusia, Grand Prix 2).

⛔ Important Warning
Never use Memory Write mode in online games. Writing to process memory can be detected by anti-cheat systems and result in a permanent ban.

Joystick Selection

Select which joystick/controller device is your H-shifter using the left dropdown.

If you want to use a clutch pedal, tick the Clutch checkbox and select the device that has the clutch axis from the right dropdown (it can be the same device as the shifter, or a separate wheel/pedal unit).

ℹ Note
Devices are listed in the order Windows enumerates them. If a device doesn't appear, make sure it is connected and recognized by Windows before launching Anyshift.

How to assign a button

  1. Click the UI button corresponding to the gear you want to assign.
  2. Engage that gear on your physical shifter.
  3. The interface will appear frozen — this is normal. Anyshift is waiting for a joystick input.
  4. Once the shifter button is detected the number updates and the UI unfreezes.
  5. Alternatively press End to cancel the assignment.
⚠ Why does the UI freeze?
This is a known limitation of how pygame (the joystick library) interacts with the GUI thread. The app is working normally — only the window appears unresponsive.

Clutch Axis & Bitepoint

Clutch Axis — the axis number on the selected clutch device that corresponds to the clutch pedal. Axis 0 is typically the first axis; check your device's axis order in Windows Game Controllers if unsure.

Bitepoint — the percentage of clutch travel required before a gear change is allowed. At 75%, the clutch must be pressed at least 75% of its range before Anyshift will register the shift. Set to 0 if you want gear changes without clutch input.

ℹ Important
Anyshift uses DirectInput scan codes, not virtual key codes. This means keystrokes are sent at the hardware level and reach the game regardless of which window has focus.

Key Delays

Press key delay — how long (in seconds) the simulated key press is held down. Default: 0.05.

Release key delay — how long Anyshift waits after releasing the key before sending the next key press. Default: 0.3.

💡 When to adjust delays
The defaults work with almost every game. If the game misses gear changes (especially rapid multi-gear shifts), try increasing the release delay to 0.4 or 0.5. If gear changes feel sluggish, try reducing it slightly.

Options

Options
☐ Detect Neutral ☐ Reverse is a button ☐ Use seventh gear ☐ Nascar mode

These four checkboxes control how Anyshift interprets your shifter input and what gear logic it applies. The correct combination depends entirely on the game you are playing.

☐ Detect Neutral

When enabled, taking the shifter out of any gear (returning to the neutral/centre position) sends the Neutral key to the game. Use this for games that support a true neutral gear, such as Grand Prix 2 or Colin McRae Rally.

❌ Leave unchecked for Gran Turismo 4 (no neutral in that game).

☐ Reverse is a button

When enabled, engaging reverse on the shifter does not up/downshift to reach reverse — it sends the Reverse key directly. Use this for games that treat reverse as a standalone button press.

✅ Enable for Gran Turismo, NASCAR Racing, IndyCar Racing.
❌ Leave unchecked for Enthusia (reverse is a sequential gear).

☐ Use seventh gear

Enable this if your shifter has a 7th gear position and you want to use it. Make sure the 7th gear button is assigned in the Joystick Buttons section — Anyshift will crash if this is enabled but no button is assigned.

☐ Nascar mode

Legacy mode for old Papyrus games (NASCAR Racing series, IndyCar Racing). In these games there is no neutral and the game remembers which gear you were in when you selected reverse. Nascar mode works around this by downshifting all the way to 1st before pressing reverse, preventing desynchronisation.

✅ Enable for NASCAR Racing, NASCAR Racing 2, IndyCar Racing.

Fields explained

FieldDescription
Active Tick this checkbox to enable memory write mode. When active, key presses are not sent — gear values are written directly to memory.
Process name The exact executable name of the running game or emulator (e.g. pcsx2.exe, dosbox.exe).
Base address The base memory address of the process. For DOSBox 0.74 this is 0x01D3A1A0. For PCSX2 you find this with Cheat Engine — no offset needed.
Memory address offset Offset from the base address to the gear variable. For PCSX2 1.6 set this to 0 and put the full address in Base address. For DOSBox the Grand Prix 2 offset is 0x291BF0.
Gear values ingame The numeric values the game stores for each gear. These vary per game. For example Enthusia uses 1–6 for gears 1–6, while reverse might be 42.

Running Anyshift

Once everything is configured, click Run Anyshift. The window will appear frozen — this is expected behaviour, not a crash.

⚠ The window freezes — this is normal
Because the joystick event loop runs on the GUI thread, the interface becomes unresponsive while Anyshift is active. Your system and the game window continue working perfectly. Only the Anyshift window is unresponsive.
  • To stop Anyshift and return to the GUI, press End on your keyboard.
  • After stopping you can change settings, load a different profile, and run again.
  • The End key works system-wide — no need to focus the Anyshift window first.

Tips & Troubleshooting

Desynchronisation

In key press mode, Anyshift and the game can occasionally get out of sync — for example on rolling starts (GT4) or after a spin. To re-sync, manually press the keyboard upshift or downshift key until the in-game gear matches your physical shifter position, then continue as normal.

To avoid desync at the start: engage the correct starting gear on your shifter before the race begins or before you gain control of the car.

Anyshift doesn't start / has no effect

  • Run Anyshift as Administrator — this is the most common fix.
  • Make sure your shifter device is connected before launching Anyshift.
  • Check that the joystick is selected correctly in the Joystick Selection dropdowns.

Game doesn't detect the key presses

  • Try increasing the Release key delay to 0.4 or 0.5.
  • Make sure the key in Anyshift matches exactly the key bound in the game.
  • Some games require the game window to be in focus. Anyshift uses DirectInput scan codes which bypass this for most games, but it may still affect a few older titles.

Clutch not working

  • Verify the Clutch Axis number. Open Windows Game Controllers and check which axis moves when you press the clutch pedal.
  • Lower the Bitepoint value if gear changes aren't registering even when the clutch is pressed all the way.
